Ти аналізуєш скан історичного архівного документу з генеалогічною інформацією.
Документ може бути метрикою, актом, списком або іншим архівним записом.

{json_shape}

Спільні правила:
  • Прізвища розпізнавати максимально точно — це головне завдання
  • Усі значення адаптувати до сучасної української мови
  • Дані брати з контексту документа; якщо є підписані поля бланку — співставляти значення
    з тим полем, якому воно відповідає, а не з випадковим текстом на полях сторінки
  • Якщо поле невідоме — null, не вигадувати
  • Усі дати в JSON (birth_date, document_date, marriage_date тощо) — **лише ISO 8601**:
    календарна дата рядком **YYYY-MM-DD**. Якщо з документа відомий лише рік — не вигадуй місяць і день:
    у полі дати **null**, рік винеси в числове поле (yob, document_year тощо), якщо воно передбачене форматом.
  • Поверни ТІЛЬКИ валідний JSON у зазначеному нижче форматі (див. блок «Формат відповіді»), без пояснень і markdown

  • Акти та метрики шлюбу: якщо для однієї особи (частіше нареченої) є і дівоче
    (до шлюбу) прізвище, і прізвище після шлюбу — поверни ДВА окремі об'єкти в масиві persons, ніби це
    два записи для пошуку: поля name, father, yob, birth_date, location однакові, відрізняється
    лише surname. Якщо ім'я в тексті подано різними формами для тієї ж особи —
    у обох об'єктах одне узгоджене сучасне ім'я. Обидва прізвища обов'язкові, не
    об'єднуй в один об'єкт. Аналогічно, якщо для нареченого згадано попереднє та нове
    прізвище.
