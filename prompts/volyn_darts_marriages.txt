Специфіка поточного набору: скани актів шлюбу (метрик шлюбу) — типовий двосторінковий бланк УРСР з таблицею «Він» / «Вона». У правій половині зображення якраз картка. Вона нас і цікавить.

На верхньому рівні особи лише ідентифікація для пошуку:
  • "surname" — завжди прізвище ДО шлюбу з офіційного рядка на кшталт «Прізвище до шлюбу»
    у колонці «Він» / «Вона» (для нареченої — дівоче).
  • "name" — ім'я, сучасна українська.

Вкладений об'єкт "meta" у "scan" (для всього скану):
  • "record" — номер акту
  • "marriage_date" — дата шлюбу лише **ISO 8601** (YYYY-MM-DD); інакше null
  • "marriage_place" — місце реєстрації шлюбу; інакше null

Усі інші відомості про особу — всередині вкладеного об'єкта "meta":
  • "surname_after_marriage" — прізвище після шлюбу лише з офіційного рядка «Прізвище після шлюбу»;
    якщо в обох колонках однакове — для кожної особи те саме; інакше null.
  • "father" — ім'я батька; null якщо невідомо. З "по-батькові" конвертувати в звичайне ім'я. Наприклад не "Григорович", а "Григорій"
  • "yob" — рік народження (ціле), лише якщо явно з тексту; інакше null.
  • "birth_date" — дата народження лише **ISO 8601** (YYYY-MM-DD), якщо з документа відомі день і місяць (або можна однозначно відновити); якщо лише рік — null і використовуй yob.
  • "birth_place" — місце народження, сучасна українська; інакше null.
  • "marriage_ordinal" — який це шлюб за рахунком для цієї особи (1, 2, …); інакше null.
  • "children_count" — кількість дітей у акті (ціле); якщо немає або не згадується — 0.
  • "etnicity" - графа національність
  • "occupation" - професія чи заняття

Не брати прізвища з підписів, приміток свідків тощо — лише з друкованих рядків таблиці.

Формат відповіді — один JSON-об'єкт:
{
  "scan": {
    "meta": {
      "record": 34,
      "marriage_date": null,
      "marriage_place": null
    }
  },
  "persons": [
    {
      "surname": "…до шлюбу…",
      "name": "…",
      "meta": {
        "father": null,
        "birth_place": null,
        "yob": null,
        "birth_date": null,
        "surname_after_marriage": null,
        "marriage_ordinal": null,
        "children_count": 0,
        "etnicity": "українець",
        "occupation": "Домогосподарка"
      }
    }
  ]
}

У масиві persons: перший елемент — «Він», другий — «Вона», якщо обидва є в бланку.
