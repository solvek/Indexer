Специфіка поточного набору: скани актів народження (метрик народження) — типовий бланк УРСР «Запис акта про народження» з блоком «Відомості про дитину» та таблицею «Батько» / «Мати». У правій половині зображення якраз картка запису. Вона нас і цікавить.

**Дата реєстрації ≠ дата народження.** У шапці/штампі (біля «Запис акта…», клейма ЗАГС) вказано, **коли записано акт** — це `registration_date`. **Не копіюй цей рік** у `birth_date` / `yob` дитини, навіть якщо він кілька разів зустрічається на бланку.

**У графі «Час народження» (п. 4) нерідко два різні 4-значні роки в одному рядку** — на старих бланках: спочатку (або в першій фразі) **друк/рукопис 19__** поруч з «**N число** … (місяця)» — це **рік фактичних пологів**; далі в тому ж рядку в кінці — знову **«… місяць 19__ р.»** — туди часто пишуть рік **запису в книзі** (він **збігається** з `registration_date` / штампом, але **не** є роком народження). **Правило:** `birth_date` / `yob` будуй **лише** з **дня + місяця** дитини та **того року, який у тому ж візуальному/смисловому блоці**, що й «N числа … місяця» (рік *перед* або *біля* цієї дати, а не **останній** `19__` у кінці рядка, якщо ці цифри **різні**). **Заборонено** брати для народження рік лише тому, що він **такий самий, як** у штампі/шапці, **або** що він стоїть **після** дня/місяця в **другому** друкованому «19__»; якщо є сумнів — уважно прочитай, до якого фрагмента (факт пологів чи «місяць 19__» в **кінці** п.4) поставлена кожна цифра. Якщо в п.4 тільки один зрозумілий день+місяць+рік **пологів** — `yob` = саме **він**, не `registration_date`.

**Відновлені / пізньо оформлені акти** — рік у штампі на роки/десятиліття **новіший** за `yob`; це **нормально**. Друковані шаблони «19__» + рукопис: **який** шаблон доповнює **дата «N числа…»** — той рік у `yob` / `birth_date` (а не сусідний шаблон в кінці рядка).

На верхньому рівні для кожної особи в масиві persons:
  • "surname" — прізвище з відповідної графи бланку (для дитини — з рядка «Прізвище» у блоці про дитину; для батька/матері — з колонок таблиці про батьків).
  • "name" — ім'я, сучасна українська.

Вкладений об'єкт "meta" у "scan" (для всього скану):
  Немає

Усі інші відомості — у вкладеному об'єкті "meta" відповідної особи.

**Дитина** (перший елемент у persons):
  • "record" — номер акту (заголовок на кшталт «Запис акта про народження № …»)
  • "registration_date" — дата **реєстрації запису в книзі** (як у штампі/шапці, не з кінця графи «Час народження»), лише **ISO 8601**; інакше null. **Не** плутати з датою народження дитини. Читай **назву місяця** (наприклад, «31-е» зазвичай **травень/май**, а не квінь); дата має бути **календарно можливою** (квінь = 30 днів)
  • "registration_place" — місце реєстрації (ЗАГС, населений пункт), якщо явно з тексту бланку чи штампів; інакше null
  • "page" — номер сторінки з поля бланку, якщо видно (число); інакше null  
  • "gender" — стать: нормалізовано українською коротко («жін», «чол») або повні слова; інакше null
  • "birth_date" — **фактична** дата пологів лише **ISO 8601** (YYYY-MM-DD) з **п.4** разом з **роком з того ж блоку, що «N числа [місяця]»** (див. попередження про **два роки** в одному рядку п.4; **не** брати рік **з кінця** графи 4, якщо там інший, ніж біля дня народження); у шапці/штампі ці цифри **не** брати
  • "yob" — рік **фактичних** пологів (ціле), **той самий** що в `birth_date` **або** пари з «N числа [місяця]» в **п.4**; **ніколи** не встановлювати рівним року з **кінця** п.4 чи `registration_date`, якщо в **п.4** явно **інший** рік біля дня народження; інакше null
  • "birth_place" — місце народження, сучасна українська; якщо в цій частині бланку немає графи або текст нечитабельний — null
  • "birth_time" — час доби народження (рядок), лише якщо в графі вказано саме час, а не лише дату; інакше null
  • "children_in_delivery" — скільки дітей народилося за цими пологами (ціле, наприклад 1 для «один»); інакше null
  • "born_alive" — true, якщо «жив» / народжений живим; false, якщо явно мертвонароджений; інакше null
  • "mother_total_births" — скільки всього дітей народилось у матері (ціле), якщо вказано; інакше null
  • "mother_children_alive" — з них лишилося в живих (ціле), якщо вказано; інакше null
  • "birth_ordinal_for_mother" — який це за порядком народження у матері (1 для «перший», 2 для «другий» тощо); інакше null

**Батько та мати** (другий і третій елементи у persons):
  • "father" — ім'я батька цієї особи; null якщо невідомо. Брати з графи «по батькові» на бланку й конвертувати в звичайне ім'я (як у загальних правилах: «Григорович» → «Григорій»); по батькові в JSON не виводити
  • "yob" — **розрахований** рік народження батька/матері (ціле): якщо в бланку є вік у графі «Вік» (`age_years`) і відомий рік народження дитини — **yob = рік_народження_дитини − age_years**. Рік дитини брати з `birth_date` (календарний рік), а якщо `birth_date` null — з поля `yob` дитини. Якщо не вистачає віку або року народження дитини — null
  • "birth_date" — дата народження ISO, лише якщо є день і місяць; інакше null
  • "birth_place" — місце народження батька/матері, якщо є в бланку; інакше null
  • "age_years" — вік у роках на момент народження дитини (ціле), наприклад з «минуло 32 років» → 32; інакше null
  • "etnicity" — графа національність
  • "occupation" — рід занять / професія

Не брати прізвища й імена з підписів, приміток на полях поза друкованими графами — лише з призначених рядків бланку.

У масиві persons: **перший елемент — дитина**; далі батьки **у порядку колонок бланку — спочатку батько («Батько»), потім мати («Мати»)**. Якщо даних для батька або матері у бланку немає — відповідний елемент не включай (не вставляй порожніх заглушок у масив).
