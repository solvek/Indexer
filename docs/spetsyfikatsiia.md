# Специфікація проєкту «Індексатор»

Цей документ фіксує **оригінальні вимоги** до інструменту (джерело: внутрішній PDF-опис проєкту). Частина формулювань може бути застарілою щодо назв моделей або майбутніх фіч — нижче є розділ **Актуалізація щодо коду в репозиторії**.

## Мета

Індексатор — скрипт, який за допомогою ШІ обробляє зображення сканів і знаходить на них інформацію про людей (генеалогічний контекст).

## Параметри інструменту

| Параметр | Тип | Обов’язковий | Опис |
|----------|-----|--------------|------|
| **Source** | string | так | URL з Google Drive **або** шлях до базової локальної папки з файлами |
| **limit** | number | ні | Скільки максимум зображень обробити. Якщо не задано — усі доступні |
| **rewrite** | boolean | ні | Чи обробляти вже оброблені зображення. За замовчуванням **true**. Перед повторною обробкою потрібно дивитись у базі; при повторі **старі записи про людей для цього скану** мають бути очищені |
| **description** | string | ні | Додатковий опис зображень/документів — щоб модель краще витягувала потрібну інформацію |
| **model** | string | ні | Наприклад «Gemini 3 Flash»; на етапі розробки в специфікації згадувалось «Gemini 3.1 Flash-Lite» |
| **Temperature** | float | ні | Температура моделі; за замовчуванням **0.1** |
| **Files** | string | ні | Фільтр файлів для обробки. За замовчуванням — усі папки та всі файли в них. Варіанти покриття: конкретний файл; усі файли з певної папки; усі папки та файли в них (рекурсивно) |

## Вхідні дані

- Скани хостяться на **Google Drive** (загальний доступ) **або** **локально**.

## Вихідні дані

- **База даних** зберігається **локально**, поруч зі скриптом.
- Якщо файлу бази немає — його потрібно **створити** автоматично.

## Сутності

### Scan

- **folder**
- **file**
- **number** — опційно; витягується з імені файлу (наприклад саме ім’я типу `00023`, або суфікс на кшталт `_00645`). Номер витягувати «інтелектуально»; якщо не вдається — поле порожнє.
- **datetime** — мітка часу обробки.

### Person

- **surname** — прізвище  
- **name** — ім’я  
- **father** — ім’я батька  
- **yob** — рік народження  
- **location** — місцевість  

## Інформація про людей (якість)

- Зображення — скани історичних архівних документів з генеалогічною інформацією (метрики, акти, списки тощо).
- **Дуже важливо** якісно розпізнавати **прізвища**.
- Імена, прізвища, локації **адаптувати до сучасної української мови**; дані брати з контексту максимально повно.
- **Ім’я батька** — з контексту або з по батькові; по батькові переводити в ім’я (наприклад «Іванович» → «Іван»).

## Хостинг і моніторинг

- Запуск: **локально** на ПК або на **Oracle Cloud (Always Free)**.
- Має бути можливість **моніторити** процес (логи, поточний статус), у тому числі **віддалено**.

## Конфігурація та ключі

- Ключі — у файлі **`.env`**.
- **`sample.env`** — зразок для копіювання.

## Актуалізація щодо коду в репозиторії

Щоб не плутати оригінал з реалізацією:

| У специфікації (PDF) | У репозиторії зараз |
|----------------------|---------------------|
| Scan.datetime | Колонка **`processed_at`** (ISO 8601, UTC) у SQLite |
| Scan.number «інтелектуально» | Евристика у `processor.extract_number()`: останній числовий блок у stem імені файлу; тип у БД — `INTEGER` (ведучі нулі не зберігаються) |
| rewrite за замовч. true | **`--rewrite` / `--no-rewrite`** у CLI; за замовчуванням перезапис увімкнено. При `--rewrite` для вже відомого скану викликається `delete_scan()` (каскадно видаляються persons), потім новий запис |
| model (Gemini 3.x у тексті спеки) | Фактична модель задається **`DEFAULT_MODEL` у `.env`** та **`--model`**; у коді типовий fallback може відрізнятись від назв у PDF — див. `sample.env` / `indexer.py` |
| Files (фільтр) | Реалізовано як **`--files`**: конкретний файл, `Папка/` (без підпапок), `Папка/**` (рекурсивно); без аргументу — усі файли рекурсивно |
| Моніторинг | Лог у консоль і **`indexer.log`**; статистика через SQL до **`indexer.db`** (див. README) |
| Telegram-бот | **Не реалізовано** — залишається roadmap |

Якщо оновите PDF або зміните вимоги, варто синхронізувати цей файл і таблицю відмінностей вище.
