# Промпти для моделі

Утиліта збирає **повний текст запиту до Gemini** з двох частин: **базового** та **розширеного** промпту.

## Базовий промпт (обов’язковий)

- **Файл у репозиторії:** `prompts/base_prompt.txt`
- **Роль:** загальні правила для **кожного** запуску індексатора: що саме витягувати зі скану (структура JSON, поля, мова, заборона на вигадування тощо).
- Змінюється разом із логікою збереження в БД: якщо змінити формат відповіді моделі, потрібно узгодити це з кодом і схемою даних.

Шаблон містить плейсхолдер `{extra}` — туди підставляється розширена частина (або порожній рядок, якщо розширений промпт не заданий).

## Розширений промпт (необов’язковий)

- **Роль:** уточнення **для конкретного набору сканів**: тип книги, регіон, особливості верстки, додаткові правила витягування. Допомагає моделі краще орієнтуватися, не змінюючи загальну схему виводу.
- **Задання з CLI:** параметр **`--extended-prompt`** (коротка форма немає; збережено **`--description`** як псевдонім для сумісності зі старими скриптами).

Значення параметра інтерпретується так:

1. **Якщо рядок складається лише з латинських літер, цифр, `_` та `-`** (наприклад `volyn_darts_marriages`) — утиліта шукає файл **`prompts/<цей_рядок>.txt`**. Якщо файл існує, його вміст додається до промпту. Якщо файлу немає, той самий рядок передається як довільний текст: `Додатковий контекст: <рядок>`.
2. **Усі інші рядки** (пробіли, кирилиця, коми тощо) — передаються як один блок довільного тексту: `Додатковий контекст: <ваш текст>`.

Приклади файлів у `prompts/`: `volyn_darts_marriages.txt`, `notarius_ros_imperia.txt` — їх можна вказати як `--extended-prompt volyn_darts_marriages` без суфікса `.txt`.
