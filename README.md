# Індексатор архівних документів

Скрипт автоматично обробляє скани архівних документів (метрики, акти, списки) через Gemini AI та зберігає знайдені генеалогічні дані в локальну SQLite базу.

## Типовий воркфлоу

1. Налаштувати ключ до Gemini API (див. нижче).
2. Скачати через «[Справну Качку](https://chromewebstore.google.com/detail/%D1%81%D0%BF%D1%80%D0%B0%D0%B2%D0%BD%D0%B0-%D0%BA%D0%B0%D1%87%D0%BA%D0%B0/gldlgeliohimejlfpgihbplkchibadim?pli=1)» потрібні справи.
3. Запустити скрипт на скачаних справах (можна одразу кілька папок зі справами).

## Залежності та `.env`

Потрібен **Python 3.9+**, віртуальне середовище та `pip install -r requirements.txt` — покроково в [документації з встановлення](docs/installation.md).

Для роботи **лише з локальними файлами** достатньо:

```bash
cp sample.env .env
```

У `.env` обов’язково вкажіть **`GEMINI_API_KEY`** ([отримати ключ](https://aistudio.google.com/apikey)). За бажанням змініть **`DEFAULT_MODEL`** — інакше лишайте значення з `sample.env`.

Змінні Google Drive у `.env` потрібні лише якщо джерело — папка в Drive; для локальних шляхів їх можна не заповнювати.

## Запуск (базові приклади)

Форма команди:

```bash
python indexer.py DBNAME SOURCE [опції]
```

`DBNAME` — ім’я або шлях до SQLite (відносні імена потрапляють у `data/`, за потреби додається `.db`). `SOURCE` — **локальна** папка зі сканами.

```bash
# Уся папка рекурсивно (база: data/my_project.db)
python indexer.py my_project /шлях/до/сканів

# Обмежити кількість нових обробок і дати моделі контекст
python indexer.py my_project /шлях/до/сканів --limit 20 --description "метричні книги, регіон"

# Один файл у підпапці
python indexer.py my_project /шлях/до/сканів --files "Справа_12/scan_001.jpg"

# З опційним експортом у CSV (див. docs/csv-export.md)
python indexer.py my_project /шлях/до/сканів --csv
```

Повні приклади (у тому числі Google Drive, фільтр `--files`) — у [використанні](docs/usage.md).

## Параметри запуску

| Параметр | За замовч. | Опис |
|---|---|---|
| `dbname` | — | Ім’я або шлях до SQLite; відносні шляхи → `data/…` |
| `source` | — | Локальний каталог із зображеннями (або URL Drive — див. доки) |
| `--files` | усі рекурсивно | Фільтр: файл, `Папка/`, `Папка/**` |
| `--limit` | без ліміту | Скільки сканів обробити (уже проіндексовані без `--rewrite` у ліміт не входять) |
| `--rewrite` / `--no-rewrite` | без перезапису | Переобробити вже відомі файли |
| `--description` | — | Текстовий контекст для моделі (регіон, тип книги) |
| `--model` | з `.env` | Модель Gemini |
| `--temperature` | `0.1` | Температура 0.0–1.0 |
| `--csv` | вимкнено | Експорт у CSV; деталі — [docs/csv-export.md](docs/csv-export.md) |
| `--verbose` | вимкнено | Детальні логи |
